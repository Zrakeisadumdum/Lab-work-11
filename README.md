# Лаб 11 - Log4j ашиглан debug хийх
## Төслийн тайлбар
Энэхүү лабораторийн ажилд Log4j 2 санг ашиглан банкны дансны үйл ажиллагааг логлож, 
Customer классын алдааг debug хийх дадлага хийсэн.
## Ангиудын тайлбар

### BankAccount
- Дансны үлдэгдлийг хадгалж, хадгалах, татах үйлдлүүдийг гүйцэтгэдэг
- Бүх 6 log level (TRACE, DEBUG, INFO, WARN, ERROR, FATAL) ашигласан
- Сөрөг утга, хэт их мөнгө татах, тэг утга зэрэг тохиолдлуудад зохих лог түвшинг ашигласан
### Customer
- Хэрэглэгчийн нэр, email хаягийг хадгалдаг
- Email-ээс домэйн нэрийг гаргаж авдаг
- Алдаа засахаас өмнө: email null үед NullPointerException үүсгэдэг байсан
- Алдаа зассаны дараа: null болон буруу форматтай email-ийг шалгаж, WARN лог гаргадаг
## log4j2.xml тохируулга
### Console Appender
- Формат: `%d{yyyy-MM-dd HH:mm:ss} [%t] %-5level %c{1}:%L - %msg%n`
- Бүх лог мессежийг консолд хэвлэнэ
### RollingFile Appender
- Файлын зам: `logs/app.log`
- Өдөр бүр шинэ файл үүсгэнэ (`TimeBasedTriggeringPolicy`)
- 1 MB тутамд файл солигдоно (`SizeBasedTriggeringPolicy` - ROLL_BY_SIZE)
- Хуучин файлуудыг gzip-ээр компресс хийнэ
### Root Logger
- Түвшин: DEBUG (бүх лог мессежийг харна)
## Алхам 2.1 - 6 log level-ийн баталгаа
Дараах тохиолдлуудад бүх 6 түвшний лог үүсдэг:
1. **TRACE**: Метод руу орох/гарах (deposit, withdraw, getBalance)
2. **DEBUG**: Үлдэгдэлийн өмнөх/дараах утга
3. **INFO**: Амжилттай гүйлгээ
4. **WARN**: Сөрөг утга, тэг утга оруулсан
5. **ERROR**: Үлдэгдэлээс их мөнгө авах
6. **FATAL**: Сөрөг утгатай данс үүсгэх
## Алхам 4.1 - tail -f-ийн давуу тал
`tail -f` нь файлын төгсгөлд шинэ мөр нэмэгдэх үед тэр даруй харуулдаг тул 
бодит цаг хугацаанд лог урсгалыг хянах боломжтой болгодог. Харин `cat` болон 
`less` командууд нь файлын одоо байгаа агуулгыг л харуулдаг, шинэчлэлтийг 
автоматаар хянадаггүй. Debug хийх үед програмын гүйцэтгэлийг дагаж лог 
гаргах нь чухал бөгөөд `tail -f` нь алдаа гарч буй мөчийг шууд харах 
боломж олгодог.
## Алдааг логоор хэрхэн олсон, зассан тухай
1. Customer классын getDomain() методод TRACE болон DEBUG логууд нэмсэн
2. Програмыг ажиллуулахад "Entering getDomain()" лог гарч, дараа нь 
   NullPointerException гарсан
3. DEBUG лог дээр email-ийн утга "null" гэж харагдсан
4. Алдаа нь email null үед substring() дуудахад үүсч байгааг тогтоосон
5. Засвар: email-ийн утгыг шалгаж, null болон буруу форматтай тохиолдлуудад 
   зохих WARN лог гаргаж, аюулгүй утга буцаахаар өөрчилсөн
